# logs/

Test fixtures and manifests for the ext4recover v5 test suite.

Test logs are runtime artifacts generated by `bash tests/<test>.sh` and
should not be committed. See `CHANGELOG.md` for the evidence each test
produced during development.

## Manifest format (`manifests/`)

```
<inode_number>|<size_bytes>|<md5sum>|<original_path>
```

Captured by `record_file` in `tests/lib_recover_test.sh` before deletion.
Each manifest serves as a test fixture — the test script compares recovered
file md5sums against these expected values.

## Test environment

Tests run on a Tencent Cloud VM with:
- `/dev/vdb` — 10 TB partitioned (test target)
- `/dev/vdc` — 10 TB work disk

See `docs/test-environment.md` for setup details.
